# 开发指南

## 架构概览

Principia 采用现代化的前后端分离架构，由基于 React 的前端和基于 Flask 的后端组成。

### 前端 (`/principia`)
使用 **Vite + React + TypeScript** 构建。

-   **核心组件**:
    -   `App.tsx`: 主应用程序编排器，管理内容、布局和 API 交互的状态。
    -   `Editor.tsx`: 封装 Monaco Editor，提供强大的代码编辑体验（支持 LaTeX/Markdown 语法高亮）。
    -   `Renderer.tsx`: 处理实时预览，使用 `react-markdown` 和 `react-latex-next`。包含自定义逻辑以在 Markdown 模式下渲染 LaTeX 颜色命令。
    -   `HandwritingCanvas.tsx`: 自定义 HTML5 Canvas 实现，支持多页手写和橡皮擦逻辑。
    -   `InteractiveBadge.tsx`: 用于显示 AI 分析结果和托管交互式可视化 iframe 的 UI 组件。
    -   `SettingsSidebar.tsx`: 管理用户对 API 端点和密钥的配置。

-   **状态管理**:
    -   主要使用本地组件状态 (`useState`) 控制 UI 切换。
    -   大量使用 Ref Hooks (`useRef`) 处理 Canvas 和 Editor 的交互以保证高性能。

### 后端 (`app.py`)
使用 **Flask** 构建。

-   **API 端点**:
    -   `/api/analyze` (POST): 处理"推理"请求。调用 LLM 解释概念并生成 HTML/JS 物理模拟代码。
    -   `/api/ocr` (POST): 处理"视觉"请求。接收 base64 图片并返回 LaTeX 转录，包含特定提示词以进行颜色检测。
    -   `/api/convert` (POST): 处理格式转换（LaTeX <-> Markdown）。包含特定的提示工程以保留文档结构并翻译颜色语法（如 `\textcolor` <-> HTML 标签）。
    -   `/`: 在生产环境中服务静态前端构建文件。

## 开发环境设置

1.  **环境要求**:
    -   安装 `Node.js` 和 `Python`。
    -   建议为 Python 使用虚拟环境 (`venv`)。

2.  **前端开发**:
    ```bash
    cd principia
    npm run dev
    ```
    这将启动带有热模块替换 (HMR) 的 Vite 开发服务器。

3.  **后端开发**:
    ```bash
    python app.py
    ```
    默认在 8000 端口运行。

## 关键实现细节

### 颜色保留逻辑
本项目的一个独特功能是在格式转换过程中保留文本颜色。
-   **后端**: `/api/convert` 端点使用专门的提示指示 LLM 将 LaTeX 中的 `\textcolor{color}{text}` 映射为 Markdown 中的 `<font color="color">text</font>`（或保留兼容的 LaTeX 语法），反之亦然，不过本处的预览貌似直接tex语法就能预览。
-   **前端**: `Renderer` 组件使用正则替换逻辑，即使在 Markdown 预览模式下也能识别并渲染 `\textcolor` 命令，确保"所见即所得"的一致体验。

### AI 可视化
可视化引擎的工作原理：
1.  将上下文文本发送给 LLM。
2.  请求生成一个包含物理模拟循环 (`requestAnimationFrame`) 的独立 HTML 代码片段。
3.  将此 HTML 注入到 `InteractiveBadge` 组件内的沙盒 `iframe` 中运行。

## MCP环境配置

### Browser MCP 简介
Browser MCP 是一个MCP（Model Context Protocol）服务器和Chrome扩展，允许您使用AI应用程序（如VS Code、Claude、Cursor和Windsurf）自动化浏览器。

### 特性
- ⚡ 快速：自动化在本地机器上进行，性能更好，无网络延迟
- 🔒 私密：由于自动化在本地进行，浏览器活动保持在设备上，不会发送到远程服务器
- 👤 已登录：使用现有的浏览器配置文件，保持登录所有服务
- 🥷🏼  stealth：通过使用真实的浏览器指纹避免基本的机器人检测和验证码

### 安装和配置

#### 1. 安装Chrome扩展
- 访问 [Browser MCP 官网](https://browsermcp.io) 下载并安装Chrome扩展

#### 2. 安装MCP服务器
由于项目中的MCP使用了workspace依赖，无法直接从源码安装。建议：
- 从 [Browser MCP 官网](https://browsermcp.io) 下载预构建的MCP服务器
- 或使用npm全局安装：
  ```bash
  npm install -g @browsermcp/mcp
  ```

#### 3. 配置环境变量
- `CLIENT_PORT`: 客户端端口（默认：9001）
- `SERVER_PORT`: 服务器端口（默认：9002）

#### 4. 启动MCP服务器
```bash
# 使用全局安装的MCP服务器
mcp-server-browsermcp

# 或使用预构建的服务器
node path/to/mcp-server.js
```

#### 5. 连接浏览器
- 点击Chrome工具栏中的Browser MCP扩展图标
- 点击"Connect"按钮连接标签页

## 代码风格指南

### 前端代码风格规范

#### TypeScript/React 代码风格

1. **缩进**
   - 使用 2 个空格进行缩进
   - 避免使用制表符

2. **命名规范**
   - **组件名**: 使用 PascalCase（如 `Editor.tsx`）
   - **函数名**: 使用 camelCase（如 `handleChange`）
   - **变量名**: 使用 camelCase（如 `userName`）
   - **常量**: 使用 UPPER_SNAKE_CASE（如 `MAX_COUNT`）
   - **接口名**: 使用 PascalCase 并以 `I` 开头（如 `IEditorProps`）

3. **代码结构**
   - 每个文件只包含一个组件
   - 组件文件使用 `.tsx` 扩展名
   - 工具函数使用 `.ts` 扩展名

4. **React 最佳实践**
   - 使用函数组件和 Hooks
   - 避免使用 Class 组件
   - 使用 `useState` 管理组件状态
   - 使用 `useEffect` 处理副作用
   - 使用 `useRef` 访问 DOM 元素

5. **TypeScript 规范**
   - 为所有变量和函数添加类型注解
   - 使用接口定义复杂类型
   - 避免使用 `any` 类型
   - 使用联合类型和泛型

6. **注释**
   - 为复杂逻辑添加注释
   - 使用 JSDoc 注释函数和接口
   - 注释应该清晰明了，解释为什么而不是是什么

### 后端代码风格规范

#### Python/Flask 代码风格

1. **缩进**
   - 使用 4 个空格进行缩进
   - 避免使用制表符

2. **命名规范**
   - **模块名**: 使用小写字母和下划线（如 `api_utils`）
   - **函数名**: 使用小写字母和下划线（如 `get_user`）
   - **变量名**: 使用小写字母和下划线（如 `user_id`）
   - **类名**: 使用 PascalCase（如 `UserService`）
   - **常量**: 使用 UPPER_SNAKE_CASE（如 `MAX_RETRY`）

3. **代码结构**
   - 每个模块只包含相关功能
   - 使用函数和类组织代码
   - 避免过长的函数（建议不超过 50 行）

4. **Flask 最佳实践**
   - 使用蓝图组织路由
   - 使用环境变量管理配置
   - 避免在路由函数中编写复杂逻辑
   - 使用装饰器处理认证和授权

5. **Python 规范**
   - 遵循 PEP 8 代码风格指南
   - 使用类型提示
   - 避免使用全局变量
   - 使用上下文管理器（with 语句）

6. **注释**
   - 为复杂逻辑添加注释
   - 使用 docstring 注释函数和类
   - 注释应该清晰明了，解释为什么而不是是什么

### 代码格式化工具配置

#### 前端格式化工具

1. **ESLint**
   - 配置文件: `.eslintrc.js`
   - 规则: 遵循 Airbnb 风格指南

   ```javascript
   // .eslintrc.js
   module.exports = {
     extends: [
       'eslint:recommended',
       'plugin:@typescript-eslint/recommended',
       'plugin:react/recommended'
     ],
     parser: '@typescript-eslint/parser',
     plugins: ['@typescript-eslint', 'react'],
     rules: {
       'react/react-in-jsx-scope': 'off',
       '@typescript-eslint/no-explicit-any': 'warn'
     }
   };
   ```

2. **Prettier**
   - 配置文件: `.prettierrc`

   ```json
   {
     "semi": true,
     "trailingComma": "es5",
     "singleQuote": true,
     "printWidth": 80,
     "tabWidth": 2
   }
   ```

3. **Tailwind CSS**
   - 配置文件: `tailwind.config.js`
   - 使用 Tailwind 类名进行样式设计
   - 避免内联样式

#### 后端格式化工具

1. **Black**
   - 代码格式化工具
   - 配置文件: `pyproject.toml`

   ```toml
   [tool.black]
   line-length = 88
   target-version = ['py39']
   ```

2. **Flake8**
   - 代码检查工具
   - 配置文件: `.flake8`

   ```ini
   [flake8]
   max-line-length = 88
   extend-ignore = E203, W503
   ```

3. **isort**
   - 导入排序工具
   - 配置文件: `pyproject.toml`

   ```toml
   [tool.isort]
   profile = "black"
   ```
